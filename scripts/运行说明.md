# Cursor 工作区配置脚本运行说明

## 问题说明

如果运行脚本时遇到语法错误（如 "意外的标记 }"），这通常是由于文件编码问题导致的。

## 解决方案

### 方法1：使用正确的编码运行（推荐）

在 PowerShell 中运行以下命令：

```powershell
# 设置控制台编码为 UTF-8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$PSDefaultParameterValues['*:Encoding'] = 'utf8'

# 运行脚本
.\scripts\setup-cursor-workspace.ps1
```

### 方法2：使用 dot-sourcing 方式运行

```powershell
# 直接加载并执行脚本
. .\scripts\setup-cursor-workspace.ps1
```

### 方法3：重新保存文件为 UTF-8 with BOM

1. 用记事本打开 `scripts\setup-cursor-workspace.ps1`
2. 选择 "另存为"
3. 在编码下拉菜单中选择 "UTF-8"
4. 保存文件
5. 重新运行脚本

### 方法4：使用 VS Code 或其他编辑器

1. 用 VS Code 打开脚本文件
2. 点击右下角的编码显示（如 "UTF-8"）
3. 选择 "通过编码保存"
4. 选择 "UTF-8 with BOM"
5. 保存文件
6. 重新运行脚本

## 快速测试

运行以下命令测试脚本语法：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -Command "Get-Content scripts\setup-cursor-workspace.ps1 -Encoding UTF8 | Out-String | Invoke-Expression"
```

## 如果问题仍然存在

如果以上方法都无法解决问题，可以：

1. 检查 PowerShell 版本（需要 5.1 或更高版本）
2. 尝试在 PowerShell ISE 中运行
3. 联系技术支持或查看 [Cursor配置说明.md](../docs/Cursor配置说明.md) 了解手动配置方法
