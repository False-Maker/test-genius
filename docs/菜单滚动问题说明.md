# 菜单滚动问题说明

## 问题描述

由于新增了多个功能模块，左侧菜单项数量增加，菜单内容超出了可视区域，需要**向下滚动**才能看到所有菜单项。

### ✅ 已优化菜单样式

已为菜单添加了滚动功能，现在菜单可以正常滚动查看所有菜单项。

## 如何查看新功能

### 方法1：向下滚动菜单

1. 将鼠标移到左侧菜单区域
2. **向下滚动鼠标滚轮**，或
3. 点击并拖动菜单右侧的滚动条（灰色细条）
4. 滚动到底部可以看到"测试评估"和"测试执行"分组

### 方法2：直接访问URL

在浏览器地址栏直接输入以下地址：

- **测试报告**：`http://localhost:3000/test-report`
- **覆盖率分析**：`http://localhost:3000/test-coverage`
- **风险评估**：`http://localhost:3000/test-risk-assessment`
- **测试规约**：`http://localhost:3000/test-specification`
- **页面元素**：`http://localhost:3000/page-element`
- **UI脚本模板**：`http://localhost:3000/ui-script-template`

### 方法3：使用浏览器搜索功能

1. 按 `Ctrl + F` 打开搜索框
2. 输入"测试评估"或"测试执行"
3. 浏览器会高亮显示对应的菜单项

## 菜单滚动条说明

菜单右侧现在有一个**灰色滚动条**：
- **滚动条位置**：菜单右侧边缘
- **滚动条宽度**：6px（较细，可能不太明显）
- **滚动条颜色**：深灰色（#4b5563）
- **悬停颜色**：浅灰色（#6b7280）

如果看不到滚动条，可能是因为：
1. 菜单内容没有超出可视区域（窗口足够大）
2. 滚动条颜色与背景色接近（已优化为更明显的颜色）

## 常见问题

### Q1: 为什么看不到滚动条？

**A:** 滚动条可能因为以下原因不明显：
- 滚动条宽度只有6px，比较细
- 滚动条颜色与背景色接近
- 如果菜单内容没有超出可视区域，滚动条不会显示

**解决方法：**
- 尝试向下滚动鼠标滚轮
- 或者直接访问URL（见方法2）

### Q2: 滚动后还是看不到新功能？

**A:** 请检查：
1. 是否滚动到了最底部
2. 浏览器窗口是否足够高
3. 菜单是否被折叠（点击左上角折叠按钮展开）

### Q3: 菜单项太多，能否优化布局？

**A:** 可以考虑：
1. 使用菜单折叠功能（点击左上角按钮）
2. 将部分菜单项合并到子菜单
3. 调整浏览器窗口高度

## 技术说明

菜单使用了以下CSS属性实现滚动：
- `overflow-y: auto` - 垂直方向自动显示滚动条
- `overflow-x: hidden` - 水平方向隐藏溢出内容
- 自定义滚动条样式，使其更明显

菜单容器高度为 `100vh - 64px`（减去顶部logo的高度），如果菜单内容超过这个高度，会自动显示滚动条。
