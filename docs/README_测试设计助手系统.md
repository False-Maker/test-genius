## 系统概述

### 系统定位
测试设计助手系统是为保险行业个险核心系统测试团队开发的智能化测试设计工具，旨在提升测试设计效率和质量，实现测试设计的标准化、智能化和自动化。

### 核心功能
系统包含两个核心模块：

1. **标准化测试设计流程模块**：
   - 建设标准化的测试设计工作台
   - 实现测试设计的全面线上化
   - 支持测试分层和测试方法选择
   - 提供测试用例管理和复用功能

2. **提示词工程用例生成模块**：
   - 基于大语言模型和提示词工程
   - 自动生成高质量测试用例
   - 支持多模型切换和对比
   - 提供用例质量评估和优化建议

### 技术架构
- **架构模式**：基于MaaS大模型应用开发平台的分层架构
- **技术栈**：Java + Spring Boot（核心业务）+ Python + FastAPI（AI能力）
- **数据库**：PostgreSQL（主数据库）+ Milvus/PGVector（向量数据库）+ Neo4j（图数据库）
- **前端**：Vue 3 + Element Plus + TypeScript
- **AI能力**：LangChain + 多模型支持（DeepSeek、豆包、Kimi、千问等）

### 实施周期
预计总实施周期：9-10个月
- 准备与规划：1个月
- 基础平台搭建：2-3个月
- 核心功能开发：3-4个月
- 测试与优化：2-3个月
- 上线与运营：持续

## 与个险核心系统的关系

测试设计助手系统与个险核心系统是独立但关联的系统：

1. **数据关联**：
   - 从个险核心系统获取业务需求和业务规则
   - 生成的测试用例可以导入到测试管理系统
   - 用例执行结果可以关联到缺陷管理系统

2. **业务关联**：
   - 针对个险核心系统的各业务模块生成测试用例
   - 支持投保、核保、理赔、保全等业务模块的测试设计
   - 了解个险核心系统的业务特点，优化用例生成

3. **技术关联**：
   - 与个险核心系统技术栈保持一致（Java + Spring Boot）
   - 共享基础设施资源（如数据库、消息队列等）
   - 遵循统一的技术规范和安全标准
