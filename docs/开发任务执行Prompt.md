# 测试设计助手系统 - 开发任务执行 Prompt

## 核心原则
以 `docs/开发进度报告与路线图.md` 作为**唯一权威路线图**，按任务清单逐步开发。每完成一个任务，必须更新路线图文档，标注完成状态和进度百分比。

---

## 工作流程（4步法）

### 第一步：任务选择与理解
1. **读取路线图文档**：`docs/开发进度报告与路线图.md`
2. **选择任务**：找到下一个未完成的任务（标记为 `[ ]` 或未标记）
3. **提取任务信息**：
   - 任务名称、工作量、所属阶段
   - 所有子任务清单
   - 依赖关系（参考文档"关键路径分析"部分）

### 第二步：代码与文档检索（必须执行）
在开始开发前，检索以下内容：

**使用工具检索**：
- `codebase_search` - 搜索相关功能实现
- `grep` - 查找相关类、方法、TODO标记
- `read_file` - 读取相关文件

**必须检索的文件类型**：
- 实体类（Entity）- 了解数据模型
- Repository接口 - 了解数据访问方法
- Service接口和实现 - 了解业务逻辑现状
- Controller类 - 了解API接口定义
- 前端页面组件 - 了解UI现状
- 配置文件 - 了解配置情况
- 相关设计文档 - 了解设计意图

### 第三步：开发实施
1. **实现功能**：
   - 遵循项目现有代码风格
   - 参考已有代码实现方式
   - 添加必要注释和日志
   - 实现异常处理和验证

2. **代码规范**：
   - **Java**：Spring Boot最佳实践、统一异常处理、日志记录
   - **Python**：FastAPI最佳实践、Pydantic验证、错误处理
   - **前端**：Vue 3 Composition API、TypeScript类型、Element Plus组件

3. **测试验证**：
   - 功能测试
   - 前后端联调（如涉及）
   - 边界情况测试

### 第四步：文档更新（必须执行）
**开发完成后，立即更新路线图文档**：

1. **更新任务状态**：
   ```markdown
   ✅ [已完成 100%] - 全部完成
   🔄 [进行中 60%] - 部分完成
   ⏸️ [已阻塞：原因] - 遇到阻塞
   ```

2. **更新子任务状态**：
   - 在每个子任务前添加 `✅` 或 `⏸️`

3. **添加完成说明**（可选）：
   ```markdown
   **完成说明**：
   - 实现位置：文件路径
   - 注意事项：重要说明
   ```

4. **更新统计信息**：
   - 更新文档末尾的"代码统计"表格
   - 更新"TODO清单统计"
   - 更新"项目现状总结"中的完成度百分比

## 任务执行检查清单

每次执行任务前，检查：
- [ ] 已读取路线图文档，明确任务要求
- [ ] 已检索相关代码和文档
- [ ] 已理解任务依赖关系
- [ ] 已确认依赖任务已完成（如需要）

每次完成任务后，检查：
- [ ] 功能实现完成并测试通过
- [ ] 路线图文档已更新任务状态
- [ ] 子任务状态已更新
- [ ] 统计信息已更新
- [ ] 代码已提交（如使用版本控制）

---

## 特殊处理规则

### 1. 任务依赖
- 如果任务A依赖任务B，必须先完成B
- 未完成依赖时标注：`⚠️ [等待依赖：任务B]`

### 2. 任务阻塞
- 遇到技术难点或依赖问题时标注：`⏸️ [已阻塞：原因说明]`
- 在文档"风险点识别"部分记录

### 3. 任务调整
- 任务范围变化时，在任务项下方添加说明
- 更新预估工作量（如有变化）

## 快速使用指南

### 在 Cursor 中使用方式

**方式1：直接引用**
```
@开发进度报告与路线图.md @开发任务执行Prompt.md 

请帮我开发第1周任务1：需求管理模块完善
```

**方式2：分阶段使用**
- 第一阶段：基础功能开发
- 第二阶段：AI核心功能
- 第三阶段：高级功能
- 第四阶段：系统优化

每次只关注当前阶段的任务。

---

## 关键提醒

1. **路线图是唯一权威**：所有任务状态以路线图文档为准
2. **及时更新文档**：每完成一个任务立即更新，不要积累
3. **保持代码一致性**：参考已有代码风格，保持统一
4. **测试驱动开发**：实现后必须测试验证
5. **文档代码同步**：代码变更后同步更新相关文档

---

## 输出要求

每次完成任务后，输出：

1. **任务完成报告**：
   - 任务名称和完成状态
   - 完成百分比
   - 实现的关键文件列表
   - 测试结果
   - 遇到的问题和解决方案

2. **文档更新内容**：
   - 路线图文档的更新位置
   - 更新的具体内容
   - 更新的统计数据
