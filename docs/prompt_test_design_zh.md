# 保险行业测试设计助手系统开发Prompt（中文版）

## 项目背景
当前项目是一个**保险行业个险核心系统**，需要为测试组开发一个**测试设计助手系统**，旨在提升测试设计效率和质量，实现测试设计的标准化、智能化和自动化。

## 核心功能需求
系统需要包含以下两个核心模块：

### 1. 标准化测试设计流程模块
- **功能定位**：建设标准化的测试设计工作台，实现测试设计的全面线上化
- **核心能力**：
  - **测试设计规范管理**：融入测试设计规范要求，确保测试设计符合行业标准和公司规范
  - **需求理解与分析**：帮助测试人员理解业务需求，识别测试要点
  - **测试分层选择**：支持测试人员根据需求选择合适的测试分层（个人、业务案例、功能案例、接口案例、场景案例）
  - **测试设计方法选择**：提供多种测试设计方法（等价类划分、边界值分析、场景法、决策表等）
  - **标准化案例分层结构**：
    - **个人级测试**：针对个人用户相关功能的测试
    - **业务案例测试**：针对完整业务流程的端到端测试
    - **功能案例测试**：针对具体功能模块的测试
    - **接口案例测试**：针对API接口的测试
    - **场景案例测试**：针对特定业务场景的测试
  - **测试用例管理**：测试用例的创建、编辑、审核、版本管理
  - **测试用例复用**：支持测试用例的复用和组合
  - **测试设计工作台**：提供可视化的测试设计工作界面

### 2. 提示词工程用例生成模块
- **功能定位**：基于大语言模型和提示词工程，自动生成高质量测试用例
- **核心能力**：
  - **提示词模板库**：基于业界测试知识和保险行业特点，构建提示词模板库
  - **智能用例生成**：
    - 根据需求文档自动生成测试用例
    - 基于测试分层结构生成对应层级的测试用例
    - 支持不同测试设计方法的用例生成
  - **公司驱动模型**：通过不同公司（保险公司）的业务特点驱动模型，生成符合特定公司业务场景的测试用例
  - **业界测试知识库**：集成业界测试最佳实践和保险行业测试经验
  - **用例质量评估**：对生成的测试用例进行质量评估和优化建议
  - **用例生成工作流**：
    - 需求输入 → 需求分析 → 测试分层选择 → 提示词生成 → 用例生成 → 用例审核 → 用例入库
  - **多模型支持**：支持调用不同的大语言模型（DeepSeek、豆包、Kimi、千问等）

## 系统架构要求

### 平台架构层次
系统应基于**MaaS大模型应用开发平台**架构，包含以下层次：

#### 1. 应用开发层
- **测试设计工作台**：核心工作界面
  - 需求管理
  - 测试设计管理
  - 用例管理
  - 权限管理
  - 测试设计规范配置
- **提示词管理平台**：管理测试用例生成相关的提示词
  - 提示词模板管理
  - 提示词版本控制
  - 提示词效果评估
  - 提示词优化
- **知识库平台**：管理测试相关的知识
  - 测试设计规范知识库
  - 保险业务知识库
  - 测试用例模板库
  - 业界测试最佳实践库
- **AI训练平台**（可选）：支持测试相关模型的微调

#### 2. 能力层
- **Agent**：智能测试设计助手代理
- **工作流**：测试设计流程编排
- **Function Call**：测试设计相关函数调用
- **MCP**：模型控制协议

#### 3. 知识库组件
- **文档管理**：测试需求文档、测试设计规范文档管理
- **文档解析**：需求文档智能解析，提取测试要点
- **知识构建**：测试知识图谱构建
- **文档检索**：智能检索测试相关知识和用例

#### 4. 模型开发层
- **模型推理**：
  - 大模型（Chat、推理类、生成类）
  - 支持调用外部模型（DeepSeek、豆包、Kimi、千问等）
- **模型微调**（可选）：
  - 测试用例生成专用模型
  - Embedding模型（用于测试用例相似度匹配）
- **模型部署**：模型服务化部署
- **模型监控**：用例生成质量监控

#### 5. 外部集成
- **外部模型**：DeepSeek、豆包、Kimi、千问等大模型
- **测试工具集成**：与测试管理工具、自动化测试工具集成

#### 6. 系统集成
- **内部系统集成**：
  - 核心业务系统（获取业务需求）
  - 需求管理系统
  - 测试管理系统
  - 缺陷管理系统

#### 7. 底层资源服务
- GPU/NPU：AI计算资源
- CPU：通用计算资源
- 网络：通信服务
- 存储：测试用例、知识库数据持久化

## 业务场景要求

### 保险行业测试特点
1. **业务复杂度高**：保险产品种类多，业务规则复杂
2. **数据敏感性**：涉及客户隐私数据，需要严格的数据安全控制
3. **合规要求严格**：需要符合保险监管要求
4. **业务流程长**：从投保到理赔的完整业务流程
5. **多系统集成**：需要与多个外部系统集成

### 测试设计场景
1. **新功能测试设计**：
   - 接收新功能需求
   - 理解业务需求
   - 选择测试分层和测试方法
   - 生成测试用例
   - 审核和优化测试用例

2. **回归测试设计**：
   - 识别变更影响范围
   - 选择需要回归的测试用例
   - 生成补充测试用例

3. **接口测试设计**：
   - 分析接口文档
   - 生成接口测试用例
   - 覆盖正常场景、异常场景、边界场景

4. **业务流程测试设计**：
   - 梳理完整业务流程
   - 生成端到端测试用例
   - 覆盖主流程、分支流程、异常流程

5. **性能测试设计**：
   - 识别性能测试场景
   - 生成性能测试用例
   - 定义性能指标和验收标准

### 用例生成场景
1. **基于需求文档生成**：
   - 输入：需求文档（Word、PDF、Markdown等格式）
   - 输出：结构化测试用例（支持多种格式导出）

2. **基于测试分层生成**：
   - 个人级用例：个人中心、个人信息管理等
   - 业务案例用例：投保流程、理赔流程等
   - 功能案例用例：核保功能、保全功能等
   - 接口案例用例：API接口测试用例
   - 场景案例用例：特定业务场景测试用例

3. **基于测试方法生成**：
   - 等价类划分法用例
   - 边界值分析法用例
   - 场景法用例
   - 决策表法用例
   - 状态转换法用例

## 技术要求
1. **大模型集成**：支持多模型切换和调用，支持模型效果对比
2. **知识库构建**：构建保险行业测试知识库和测试设计规范知识库
3. **工作流编排**：支持测试设计流程的灵活配置和自动化
4. **接口服务**：提供标准化API接口，支持与测试管理工具集成
5. **权限管理**：细粒度权限控制，支持测试用例的审核流程
6. **版本管理**：支持测试用例的版本管理和变更追踪
7. **模板管理**：支持测试用例模板的自定义和管理
8. **质量评估**：对生成的测试用例进行质量评估（覆盖率、完整性、可执行性等）

4. 知识库设计方案

## 特殊要求
1. **保险行业适配**：系统设计需要考虑保险行业的特殊性和合规要求
2. **测试规范遵循**：生成的测试用例需要符合公司测试设计规范和行业标准
3. **用例可执行性**：生成的测试用例应该具备良好的可执行性，便于自动化测试
4. **知识积累**：系统应该能够持续学习和积累测试设计经验
5. **协作支持**：支持测试团队的协作，包括用例评审、知识共享等
